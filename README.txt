Decidí usar la librería math.js para analizar expresiones más complejas y anidadas. Al intentar parsear la expresión correctamente, descubrí que necesitaba crear un árbol sintáctico para separar los diferentes tokens y traducirlos a código que JavaScript pudiera entender y operar mediante la creación de un new Function(). La librería math.js es liviana y ofrece un entorno controlado con mayor seguridad contra inyecciones maliciosas de JavaScript. Además, eval() no es seguro y new Function() es menos seguro que la librería. Teniendo en cuenta estos aspectos, creo que tiene sentido incluir la librería en el proyecto.
